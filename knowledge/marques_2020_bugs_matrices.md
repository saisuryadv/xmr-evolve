## 1. EVERY BUG MENTIONED

### Bug 1: STEXR (STEMR) Orthogonality Failure on Clustered Eigenvalues

**Location**: Appendix B, pp. 14:23-14:24
**Routine**: STEXR (the Willems-Lang 2013 MRRR implementation)
**Description**: STEXR fails to produce orthogonal eigenvectors for a specific tridiagonal matrix T with prescribed eigenvalue distribution lambda_i = c^((i-1)/(n-1)), c = 1/sqrt(e), i=1,2,...n, n=10 (so lambda_1 ~ 1.49e-8, ..., lambda_n = 1.00).

**Specifics**:
- The eigenvalues returned by STEXR on its main diagonal agree well: ||T - Z_XR * Lambda_XR * Z_XR^T|| / (||T|| * n * eps) ~ 0.6
- But orthogonality is catastrophic: ||I - Z_XR^T * Z_XR|| / (n * eps) ~ 3.95e4
- The first four columns of Z_XR (corresponding to lambda_1 ~ 1.49e-8, lambda_2 ~ 1.10e-7, lambda_3 ~ 8.17e-7, lambda_4 ~ 6.06e-6) are linearly dependent
- In contrast, STEVX and STEMR produce orthogonality ~ 0.9 and 0.1 respectively

**Exact matrix entries**: Table B.1 gives all 10 diagonal and 9 off-diagonal entries of T (generated by DLATMS with prescribed eigenvalues, then tridiagonalized by DSYTRD).

**Root cause**: Not fully diagnosed in the paper, but attributed to STEXR's MRRR implementation failing on very close eigenvalues.

### Bug 2: STEMR NaN Safeguard Issues (Footnote 2)

**Location**: Page 14:4, Footnote 2
**Routine**: STEMR (DSTEMR in LAPACK)
**Description**: "At the time of this writing, [LAPACK 2018] lists two issues (bugs) related to STEMR. The issues are related to safeguards mechanisms against NaNs, which may lead to an early termination of STEMR even for some relatively benign matrices."
**Reference**: LAPACK GitHub Reference-LAPACK issue tracker (as of 2018)

### Bug 3: BDSCR (Willems et al. 2006) -- Coupled B^TB/BB^T MRRR Abandoned

**Location**: Page 14:3-14:4
**Routine**: BDSCR (proposed by Willems, Lang, and Voemel, 2006)
**Description**: An implementation called BDSCR that applied MRRR to the coupled eigenvalue decompositions of B^TB and BB^T "was abandoned -- unsatisfactory results (large residuals for the SVD) were obtained in a number of cases, revealing gaps in the theory for the coupled approach."
**Root cause**: "The problem was found to be related to a potential mismatch in twisted factorizations used by MRRR in that context [Willems 2010, p. 128, 149]."

### Bug 4: STEVX Known Failure Modes

**Location**: Page 14:4-14:5
**Routine**: STEVX (bisection + inverse iteration)
**Description**: Multiple known failure modes:
1. "In extreme situations of tightly clustered eigenvalues, the bisection may potentially not converge to the desired accuracy"
2. "Not all eigenvalues within an interval defined by indices may be found"
3. "Inverse iteration may fail to converge with the allowed number of iterations"
4. STEVX may "occasionally fail to deliver orthogonal eigenvectors when the eigenvalues are too closely clustered" (although STEIN implements modified Gram-Schmidt reorthogonalization)

**Note**: The authors state "our exhaustive tests with STEVX (including the ones shown in Section 5) have never detected such anomalies."

### Bug 5: BDSVDX Reorthogonalization Insufficient for CHKBD Matrices

**Location**: Pages 14:9, 14:13-14:14
**Routine**: BDSVDX
**Description**: For the CHKBD matrix B_8x8 (n=8), the reorthogonalization strategy in BDSVDX (based on checking ||u_hat|| - 1/sqrt(2) >= tol where tol = sqrt(eps)) is NOT sufficient. The orthogonality metrics orthU and orthV are O(10^5) even though the norm-based test passes.

**Specific case**: B_8x8 with a_i = 10^(-(2i-1)), b_i = 10^(-(2i-2)), i=1,2,...n, n-1. Condition number ~ O(10^22). The three smallest singular values are s_6 = O(10^-10), s_7 = O(10^-12), s_8 = O(10^-22). The norm test ||u_hat_i|| - 1/sqrt(2)| < 10*eps for all i, but the vectors are NOT orthogonal.

**Diagnosis**: The eigenvectors z_7 and z_8 of T_GK associated with eigenvalues -s_7 and -s_8 have small reciprocal condition numbers [Anderson et al. 1999, p. 103], O(10^-12). The singular vectors extracted from these eigenvectors are NOT orthogonal to machine precision, yet STEVX's eigenvectors z are orthogonal to working precision: ||I - Z^T*Z|| / (2*n*eps) = 0.125.

**Fix needed**: "The strategy for triggering reorthogonalization needs also to take into account the separation and magnitude of the eigenvalues and not only ||u_hat|| and ||v_hat|| (as in the current LAPACK implementation) or simply a tighter tol."

### Bug 6: STEXR No Longer Maintained

**Location**: Page 14:24
**Description**: "To the best of our knowledge, STEXR is no longer maintained, justifying our choice of STEVX for the first implementation of BDSVDX."

## 3. CHKBD MATRIX CONSTRUCTION -- EXACT FORMULA

**Source**: Page 14:11, described as "borrowed from the LAPACK SVD tester"

**Definition**: A bidiagonal matrix with entries defined as e^x, where x is chosen uniformly on [2*log(eps), -2*log(eps)], ranging from eps^(-2) to eps^2.

**Note**: The footnote 5 on page 14:9 states "This matrix is not included in the tests in Section 5."

**Specific CHKBD instance used in Section 4 analysis** (B_8x8, n=8):
- Diagonal entries: a_i = 10^(-(2i-1)), i = 1, 2, ..., n
- Superdiagonal entries: b_i = 10^(-(2i-2)), i = 1, 2, ..., n-1

So:
```
a = [1e-1, 1e-3, 1e-5, 1e-7, 1e-9, 1e-11, 1e-13, 1e-15]
b = [1e-0, 1e-2, 1e-4, 1e-6, 1e-8, 1e-10, 1e-12, 1e-14]
```

Properties:
- Two-norm condition number ~ 1.005
- But actual condition number ~ O(10^22) (ratio of largest to smallest singular value)
- s_1 ~ O(10^(+31/500... extrapolating)) ... paper says for n=500: s_1 ~ O(10^(+31)), s_n ~ O(10^(-284))
- Three smallest singular values: s_6 = O(10^-10), s_7 = O(10^-12), s_8 = O(10^-22)

**General CHKBD for testing** (matrices 248-250 in the testbed, the outliers):
- Dimensions: 125, 250, 500 respectively
- For n = 500: s_1 ~ O(10^(+31)) and s_n ~ O(10^(-284)) as computed by BDSQR
- resid is O(10^-2), but orthU and orthV are O(10^(+13))

## 4. EVERY EVALUATION METRIC AND THRESHOLD

### Three Primary Metrics:

1. **resid** = ||U^T * B * V - S|| / (||B|| * n * eps)
   - Measures accuracy of the decomposition
   - ||.|| is the Frobenius or 2-norm (not specified explicitly, but standard is Frobenius)
   - B is the bidiagonal matrix, n is its dimension, eps is machine precision

2. **orthU** = ||I - U^T * U|| / (n * eps)
   - Measures orthogonality of left singular vectors

3. **orthV** = ||I - V^T * V|| / (n * eps)
   - Measures orthogonality of right singular vectors

### Machine Precision Values:
- Double precision: eps ~ 1.11e-16
- Single precision: eps ~ 5.96e-8

### Plotting Conventions (for result interpretation):
- x-axis bounded by floor and ceiling: max(10^-1, resid) and min(10^4, resid)
- Similarly for orthU and orthV
- Results > 10^4 are explicitly discussed as outliers

### Classification Thresholds (from histogram bins in Figures 3 and 5):
- **[10^-1, 10^0)**: Excellent (the vast majority of results)
- **[10^0, 10^1)**: Acceptable but notable
- **[10^1, 10^2)**: Concerning
- **[10^2, 1/eps]**: Failure / outlier

The paper considers results **above 10.0** as noteworthy and results **above 100** as outliers.

### Reorthogonalization Trigger:
- tol = sqrt(eps)
- Triggered when: | ||u_hat|| - 1/sqrt(2) | >= tol (similarly for v_hat)
- This is tested per-vector after extraction from the TGK eigenvector

### Failure Modes by Algorithm:

**BDSQR (implicit QL/QR)**:
- No accuracy failures reported
- Used as the reference/baseline for singular values and accuracy comparison
- Limitation: Full SVD only, O(n^2) per sweep with O(n) sweeps typical but can be O(n^2) worst case, so O(n^3) total

**BDSDC (divide-and-conquer)**:
- No accuracy failures reported in this paper
- Limitation: Full SVD only
- Memory footprint: largest of all three, exceeds cache for larger matrices

**BDSVDX (TGK + STEVX)**:
- **Outlier 1**: Matrix 36 (dim 1,260) from Lanczos without reorthogonalization. Largest 138 eigenvalues of tridiagonal agree to 12 digits. Large clusters of singular values cause outlier behavior.
- **Outlier 2**: Matrix 225 (dim 396) with singular value distribution having large clusters. Largest 41 singular values of order O(10^-2) agree to 13 digits.
- **Outlier 3**: Matrices 248-250 (CHKBD type, dims 125, 250, 500). resid is O(10^-2) but orthU and orthV are O(10^(+13)). These are the worst outliers.
- **Outlier 4** (RANGE="I"/"V" only): Matrix 97 (case 966), dim 2,100, formed from 100 copies of Wilkinson W21+ glued with gamma = 10^-11. Highly clustered eigenvalues/singular values. Values set to VL and VU lead to computation of 1,657 singular values/vectors in case 966.
- **Refinement turned off**: For matrices 248-250 (CHKBD), the reorthogonalization strategy was turned off because the norm-based trigger never fires, yet the vectors are not orthogonal.

**BDSCR (Willems coupled MRRR)**:
- Abandoned entirely due to "unsatisfactory results (large residuals for the SVD)"
- Root cause: "potential mismatch in twisted factorizations used by MRRR"

**STEXR (Willems-Lang MRRR)**:
- Fails catastrophically on prescribed eigenvalue distribution matrix (Appendix B)
- Orthogonality degrades by factor of ~40,000 compared to STEVX/STEMR
- Also fails on glued Wilkinson matrices
- "No longer maintained"

**STEVX (bisection + inverse iteration)**:
- Theoretically can fail on tightly clustered eigenvalues (bisection non-convergence, missing eigenvalues, inverse iteration non-convergence)
- In practice: "our exhaustive tests ... have never detected such anomalies"
- Chosen for BDSVDX implementation despite theoretical risks

## 7. THE TGK APPROACH -- HOW IT DIFFERS FROM WILLEMS

### The BDSVDX TGK Approach:

1. **Form the Golub-Kahan matrix T_GK** from bidiagonal B:
   ```
   T_GK = tridiag(a_1, b_1, a_2, b_2, ..., b_{n-1}, a_n)
                   (0,   0,   0,   0, ...,  0,       0)
                   (a_1, b_1, a_2, b_2, ..., b_{n-1}, a_n)
   ```
   This is a 2n x 2n symmetric tridiagonal matrix with the perfect shuffle permutation P such that C = P * T_GK * P^T, where C is the Jordan-Wielandt matrix [0, A; A^T, 0].

2. **Compute eigenvalues/vectors using STEVX** (bisection + inverse iteration on T_GK). Target the negative eigenvalues to get singular values in descending order.

3. **Extract u and v** from eigenvectors z of T_GK:
   - u_i = odd entries of z_i (rows 1, 3, 5, ...)
   - v_i = even entries of z_i (rows 0, 2, 4, ...)
   - Normalize u_i and v_i explicitly

4. **Reorthogonalize** if necessary (Gram-Schmidt against previous vectors, triggered by norm deviation from 1/sqrt(2)).

5. **Handle splitting** at zero diagonal or superdiagonal entries, creating rectangular sub-bidiagonals with special null vector handling.

### Key Advantage Over Normal Equations (B^TB):
- Singular values are NOT squared, so small singular values maintain relative separation
- Both U and V are extracted simultaneously from T_GK eigenvectors
- Subset computation is natural (just compute selected eigenvalues of T_GK)

### How This Differs from Willems (2006, 2010, 2013):

**Willems et al. 2006 (BDSCR)** -- Coupled B^TB/BB^T approach:
- Applied MRRR to BOTH B^TB and BB^T simultaneously
- Attempted to exploit the relationship between eigenvectors of B^TB and BB^T
- Used twisted factorizations that turned out to have mismatches
- Result: ABANDONED due to large residuals

**Willems 2010 (dissertation)** -- NCD-aware approach:
- Proposed using MRRR on T_GK directly (same formulation as BDSVDX)
- Developed theory for "non-canonical deflation" (NCD) specific to TGK structure
- The Grosser-Lang algorithm (Algorithm 4.1) was partially implemented as DBDSGR but was incomplete (missing DLARRI eigenvalue refinement)

**Willems and Lang 2013 (STEXR)** -- Generic MRRR on T_GK:
- Implemented MRRR for general symmetric tridiagonal matrices
- Applied to T_GK without exploiting its special structure
- EXPOSED DEFICIENCIES: "our experiments with an implementation given in Willems and Lang [2013] and named STEXR exposed deficiencies (inaccuracy) for relatively simple matrices"
- Specifically failed on the Appendix B matrix

**BDSVDX (this paper)** -- TGK + STEVX:
- Uses the same T_GK formulation as Willems
- But uses STEVX instead of MRRR for the eigenproblem
- STEVX is more robust but O(n^2) per eigenvector in the worst case (vs O(n) for MRRR)
- Plans to replace STEVX with MRRR "when such an implementation becomes available"
- Key insight: accurate eigenpairs of T_GK are necessary but NOT sufficient -- the extraction of u and v and their orthogonality also matters

## 9. RANDOM MATRIX GENERATION METHODOLOGY

### DLATMS Usage:
- LAPACK subroutine DLATMS generates random symmetric matrices with prescribed eigenvalue distributions
- Used for the Appendix B STEXR failure case: eigenvalues lambda_i = c^((i-1)/(n-1)), c = 1/sqrt(e), n=10
- Then tridiagonalized by DSYTRD to get T
- Then Cholesky factored as T - nu*I = L*L^T to get bidiagonal B = L^T

### DLARNV Usage:
- LAPACK's DLARNV generates random vectors
- Used with two distributions:
  1. **Standard normal distribution** (IDIST=3 in DLARNV terminology)
  2. **Uniform distribution** (IDIST=1 or IDIST=2)
- Random bidiagonal matrices have entries drawn from these distributions

### CHKBD Construction for Testing:
- Entries defined as e^x where x is chosen uniformly on [2*log(eps), -2*log(eps)]
- Range: from eps^(-2) to eps^2
- This creates entries spanning a massive dynamic range

### Matrix Perturbation for RANGE="V" Tests:
- Given B, build corresponding T, compute eigenvalues via bisection (STEBZ)
- Generate n_V = 10 random pairs of indices IL, IU
- Map indices to eigenvalues, then map to values VL, VU
- **Perturb eigenvalues slightly** to obtain eigenvalues of T (to ensure VL < VU as required by STEVX)
- Invoke BDSVDX n_V times with corresponding VL, VU pairs

### Matrix Perturbation for RANGE="I" Tests:
- Generate n_I = 10 random pairs of indices IL, IU
- Invoke BDSVDX n_I times with corresponding index pairs
- "This mapping can produce values that differ from the ones obtained by simply perturbing the eigenvalues"

### Zero-Entry Injection:
- "The infrastructure also allows the setting of a percentage of entries of B that will be randomly set to 0, thus enabling the testing of a variety of splitting patterns."

## APPENDIX: Key Algorithmic Details

### TGK Matrix Construction (Equation 4):
```
T_GK = tridiag(  a_1,    b_1,   a_2,   b_2,  ..., b_{n-1}, a_n  )
               (  0,      0,     0,     0,    ...,  0,       0   )
               (  a_1,    b_1,   a_2,   b_2,  ..., b_{n-1}, a_n  )
```
Dimension 2n, with the "perfect shuffle" permutation P = [e_{n+1}, e_1, e_{n+2}, e_2, e_{n+3}, ..., e_{2n}].

### Eigenvector-to-Singular-Vector Mapping:
If z = (u^T, +/-v^T)^T / sqrt(2) is a normalized eigenvector of T_GK for eigenvalue +/-s, then:
- u (left singular vector) = odd-indexed entries of P^T * z, scaled by sqrt(2)
- v (right singular vector) = even-indexed entries of P^T * z, scaled by sqrt(2)
- Both u and v are explicitly normalized

### Splitting Criterion:
Same as BDSQR (Demmel and Kahan 1990):
- Check superdiagonal entries of T_GK with row indices 2, 4, ... (i.e., b_i entries)
- If b_i is negligible, split
- Then check diagonal entries of T_GK with row indices 1, 3, ... (i.e., a_i entries)
- If a_i is negligible, split (creating zero at top/bottom/interior)

### BDSVDX Pseudocode (Table A.1):
```
BDSVDX: input(range, n, a, b, vl, vu, il, iu), output(s, z)
form matrix T_GK: t(1:2:2n) = a, t(2:2:2n-1) = b, j_l = 1
for i = 2:2:2n
    if t(i) is negligible or i = 2n, then j_r = i
        for j = j_l:2:j_r
            if t(j) is negligible or j = j_r, then
                invoke STEVX to process active matrix
                normalize singular vectors
                apply reorthogonalization
            end
        end
    end
end
sort singular values/vectors
discard singular values/vectors depending on range
reorder array z
```

### Key Insight for Future MRRR-based Implementation:
The paper explicitly states the plan to "replace STEVX with a more robust implementation of MRRR for Equation (4) when such an implementation becomes available." The key requirements:
1. MRRR must handle T_GK's special structure (paired +/- eigenvalues, zero eigenvalue for odd-dimension blocks)
2. Must produce orthogonal eigenvectors even for tight clusters
3. Must maintain relative accuracy for small eigenvalues
4. The extracted u, v vectors need additional orthogonality guarantees beyond what T_GK eigenvector orthogonality provides
