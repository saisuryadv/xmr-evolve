How we got bit-identity (Py ↔ Fortran)

A
Compiled the orchestration with -O0 -fno-fast-math so gfortran preserves source-order floating-point evaluation. -O2 would reorder associative ops, fuse FMA, and vectorize — each introducing 1–2 ULP drift.
B
Replaced np.linalg.norm with a system BLAS DNRM2 ctypes binding. NumPy's bundled OpenBLAS DNRM2 differs from reference BLAS's DNRM2 at ~1 ULP — enough to break a 2·ε bit-match. Now both sides hit the same compiled dnrm2_ symbol.
C
Replaced np.random.randn in the GS-completion fallback with LAPACK DLARNV via ctypes, with deterministic 4-tuple seeds derived from the column index. Both Python and Fortran call the same RNG with the same seed sequence.
D
Same XMR libxmr.so linked from both paths (Python via ctypes, Fortran via direct link). Bit-identity guaranteed: every floating-point op routes through the same compiled binary symbols.
Validation  —  21,375 / 21,375 specs match within 2·ε across three corpora.
σ bit-identical on every test bidiagonal. U / V differ by ≤ 1 ULP on a handful of gl_wilkw / gl_wilk2w adversarial cases.
