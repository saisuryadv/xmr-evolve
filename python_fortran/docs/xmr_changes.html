TL;DR

The XMR kernel itself was edited in only two files, totalling ~20 Fortran lines. The bigger lift is the 478-line C wrapper we wrote from scratch — it bakes in the call-site choices (Geršgorin bounds, abserr=0, emode='o', tight per-eigenvalue brackets, gaptol = 0.02/n) that unlock the fixed kernel paths.

Fix 1 dlaxrb_clssfy_fix.f — depth-0 gap classification

A single condition was effectively dead at the root level (DEPTH = 0). Every eigenvalue pair was being classified as a "singleton with full gap" — even when it was actually inside a tight cluster. RQI then converged to the wrong root and orthogonality blew up.

The pattern (repeated at lines 361, 414, 445)

dlaxrb_clssfy.f → dlaxrb_clssfy_fix.f line 361
@@ depth-0 gap classifier @@
-      IF( SPREAD .GE. MIN( AVGTHRESH, ABSMAX*GAPTHRESH ) )THEN
+      IF( AVGTHRESH .GT. ZERO )THEN
+         GAPOK = SPREAD .GE. MIN(AVGTHRESH, ABSMAX*GAPTHRESH)
+      ELSE
+         GAPOK = SPREAD .GE. ABSMAX*GAPTHRESH
+      ENDIF
+      IF( GAPOK )THEN

⚠ Upstream behaviour at the root

At DEPTH = 0:

AVGTOL = MAX(SPDIAM, REPELG)
       * (0 · AVGAPFAC) / (N−1)
       = 0
AVGTHRESH = 0
MIN(0, ABSMAX·GAPTHRESH) = 0
SPREAD ≥ 0   ←  always true ✗

Every depth-0 pair → singleton, regardless of relative gap.

✓ Our fix

When AVGTHRESH = 0 (depth 0), skip the absolute term and fall back to the relative criterion alone:

SPREAD ≥ ABSMAX · GAPTHRESH

This is what depth-0 was always meant to do — only relative separation matters when no absolute tolerance has been chosen yet.

Why it shows up on Godunov: Godunov spectra have eigenvalue pairs separated by ~10⁻⁸ but with absolute values of order 1. The guard ABSMAX·GAPTHRESH ≈ 1·10⁻¹⁴ is the right test — but MIN(0, 10⁻¹⁴) = 0 erased it. Pairs got declared singletons, RQI walked into the wrong eigenvector, and ortho-loss exceeded 30 nε.

Fix 2 dlaxre_gk.f — GK root + tiny-E additive perturbation

Two tiny hunks (+5 net lines) make the root representation respect the TGK structure that comes out of the Golub–Kahan formulation of a bidiagonal SVD.

Hunk A — enable the GK branch (lines 269–275)

dlaxre.f → dlaxre_gk.f root-rep selector
-      IF( .FALSE. )THEN
-C        Support for GK-type matrices deactivated.
-C        Reason: One should call a specific SVD routine in this case,
-*         see below. We ran into problems for GRO-type synthetics,
-*         mainly because full blocks in the root restrict possible
-*         choices for blocks in the children too much, due to
-*         overlap sequences.
+      IF( ABS(DMAX)+ABS(DMIN) .LT. EPS*EMAX )THEN
+C        GK-type matrix: constant (zero) diagonal.
+C        Use T_GK directly as root per Algorithm 4.1 (Willems-Lang 2012).
+C        No outside shift needed — preserves GK structure.

Upstream had this branch hard-wired off. For bidiagonal SVD the input to dlaxre is always TGK by construction — zero diagonal, alternating bidiagonal entries on the off-diagonal. Forcing an outside shift on TGK destroys the alternating-zero structure and the children inherit broken blocks.

Hunk B — additive perturbation for tiny entries (line 492)

dlaxre.f → dlaxre_gk.f RRR perturbation loop
-      E(I) = E(I) * (ONE + RWORK(I)*FAC)
+      IF( ABS(E(I)) .LT. EMAX*1D-1 .AND.
+     $    FAC*EMAX .LT. ABS(E(I)) )THEN
+C        Tiny entry relative to EMAX, but additive perturbation
+C        won't overwhelm it.  Breaks periodic structure in
+C        shifted representations (GK matrices).
+         E(I) = E(I) + RWORK(I)*FAC*EMAX
+      ELSE
+         E(I) = E(I) * (ONE + RWORK(I)*FAC)
+      ENDIF

⚠ Multiplicative perturbation

For Godunov: E(i) = 2.5e-7, EMAX ≈ 1, FAC ≈ 1e-1:

E(i) · (1 + 0.1) = 2.75e-7

The perturbation is invisible — the boundary survives but won't move under shifted representations.

For genuinely tiny entries near rounding floor: (1 + δ) · 1e-15 ≈ 1e-15, effectively no perturbation at all → RRR not robust.

✓ Additive perturbation

E(i) + δ · EMAX = 2.5e-7 + 1e-14

Perturbation is small vs EMAX but large vs E(i), which is exactly the regime where the RRR robustness criterion bites. The block boundary that 2.5e-7 encodes survives all downstream shifted children.

Reverting this patch alone causes 146/379 evaluations to fail with ortho-blowup of order 10¹¹ nε (per docs/02_xmr_modifications.md).

Fix 3 xmr_wrapper.c — entirely new C orchestration

Upstream initial_code/ has no C wrapper at all — the kernel was driven from a top-level Fortran orchestrator (dstexr.f) that we dropped for being unreachable. We replaced it with a 478-line C wrapper that owns every call-site choice the kernel exposes.

The wrapper isn't glue. It encodes the algorithmic policy for how the bidiagonal-SVD driver invokes the per-block eigensolver. Three call sites carry the load:

3a. xmr_eigenvectors() — drives dlaxre + dlaxrv per block

xmr_wrapper.c lines 303–320
  double gl = evals_in[0]   - 0.01 * |evals_in[0]|   - 1e-20;
  double gu = evals_in[n-1] + 0.01 * |evals_in[n-1]| + 1e-20;
  double abserr = 0.0;
  char   emode  = 'o';     // outer Geršgorin path inside dlaxre
  dlaxre_(&n, d_arr, e_arr, &gl, &gu, &abserr, &gaptol,
          repr, repi, &tau_re,
          ewl_ae, ewl_lu, &emode, rwork_re, &info_re, 1);

The Fortran mirror lives at mr3gk_fortran/mr3gk_tgk.f90:75–86 for the pure-Fortran path. Both paths agree on the same four choices:

ArgumentChoiceEffect on kernel
gl, gu ±1% Geršgorin around the bisection-derived extremes, with a 1e-20 floor Tighter than the bound a naïve caller would pass; first bisection in dlaxre converges in a few steps instead of ~30.
abserr 0.0 Forces AVGTHRESH = 0 at depth 0 — which is the precondition that makes Fix 1's relative-only guard fire.
emode 'o' (outer) Selects the outer-Geršgorin code path inside dlaxre that computes the bracket from the RRR itself rather than the original T. Bracket is much tighter for graded bidiagonals.
gaptol max(GAPTOL, 0.02 / n_nonneg) 10–1000× tighter than LAPACK default on n ≥ 100. Borderline pairs are routed to CLUSTER (recurse) instead of SINGLETON (premature RQI).

3b. xmr_find_child_repr() — tight per-eigenvalue brackets

xmr_wrapper.c lines 197–208
  for (int j = 0; j < iclen; j++) {
    int    i  = icbeg + j;
    double ev = evals[j];
    double err = n * 2.22e-16 * fabs(ev);   // n·ε·|λ|
    if (err < 2.22e-16) err = 2.22e-16;    // floor at ε
    fewl_lu[2*i - 2] = ev - err;            // EWL lower
    fewl_lu[2*i - 1] = ev + err;            // EWL upper
    fewl_ae[2*i - 2] = icbeg; fewl_ae[2*i - 1] = icend;
  }

The EWL (eigenvalue list) is what dlaxrf uses to pick shifts for cluster children. A loose bracket here forces dlaxrf to re-bisect; an overtight bracket can skip a real eigenvalue. n·ε·|λ| with an ε floor is the right scale — it tracks the actual conditioning of the cluster.

3c. frginfo setup — explicit "full gap" at cluster edges

xmr_wrapper.c lines 189–191
  frginfo[0]       = 2;   // left boundary:  FULL gap
  for (i=1; i<iclen; i++) frginfo[i] = 0;  // interior: NO gap
  frginfo[iclen]   = 2;   // right boundary: FULL gap

Tells dlaxrf that the cluster boundaries are hard (full gaps) but the interior is contiguous. Without this, dlaxrf can pick shift candidates that cross the boundary and break orthogonality with neighbour clusters.

What we removed (and why none of it was on the critical path)

5 size-variants unreachable

dlaxrm_stat4 / 8 / 16 / 32 / 64 — statistics modules parameterised by MAXPARNEG. dlaxrm.f hard-codes MAXPARNEG = 2 at compile time, so only stat2 is ever entered.

5 algorithm variants superseded

dlaxra, dlaxri, dlaxrk, dlaxrn0, dlaxro — preprocessing, isolated-root builder, alternate bisection, n=0 branch, eigvec reordering. None are reached from our wrapper's call graph.

Top-level driver replaced

dstexr.f (657 lines) is upstream's Fortran orchestrator. Our C wrapper and mr3gk_fortran/ replace it with a thinner, direct call to dlaxre + dlaxrv.

1 outdated dlaxre overridden

Upstream dlaxre.f is replaced at link time by dlaxre_gk.f (Fix 2) sitting one directory up.

Reporting infra research-only

stats.f (1322 lines of common-block timing hooks) and reptools.f — instrumentation for the original research codebase, not needed for production SVD.

README.md obsolete

Documented files that no longer exist in our subset.

Why these three changes compose

Each change addresses a distinct failure mode of upstream MR³ on graded/clustered bidiagonals. The composition matters more than any one fix:

① Classification (Fix 1)

What broke: root cluster pairs classified as singletons.

Why it matters downstream: a misclassified singleton sends its eigenvalue straight to RQI before dlaxrf ever gets to refine the cluster's shared shift.

② Representation (Fix 2)

What broke: GK structure destroyed by upstream's mandatory shift, then tiny off-diagonals erased by multiplicative perturbation.

Why it matters downstream: children of the root inherit a representation that isn't relatively robust — RRR fails its core invariant.

③ Call-site policy (Fix 3)

What broke: nothing in upstream — but the right defaults aren't obvious. Loose bounds, wrong emode, slack gaptol, sloppy EWL brackets — any one of these breaks the chain.

Why it matters downstream: bad seeds in dlaxrv can't be saved by correct classification or robust RRR.

The chain: Wrapper sets up tight brackets & tight gaptol → Fix 1 classifies clusters correctly at depth 0 → Fix 2 builds a GK-aware, robustly-perturbed RRR for each cluster → dlaxrf picks good shifts for children → dlaxrv applies RQI to genuinely isolated eigenvalues only. Break any link and the orthogonality budget collapses by 4–6 orders of magnitude.
