# Pure-Fortran MR3-GK SVD — Standalone evaluation, 2026-05-09

This document records the result of running the project's **standard 379-test
evaluation harness** (`evaluate.py`, the same one used to score the
Python+Fortran hybrid in `README.md`) routed through the **pure-Fortran
driver** `mr3gk_fortran/mr3gk_run`.

**Result: 379 / 379 passed (100 %).**

## How the run was performed

```bash
cd python_fortran

# 1. Build libxmr.so (the XMR kernel + our two patches)
bash build.sh

# 2. Build the pure-Fortran orchestration driver
cd mr3gk_fortran && bash build.sh && cd ..
# produces ./mr3gk_fortran/mr3gk_run

# 3. Route the 379-spec evaluation harness through the Fortran driver
python3 run_evaluate_fortran.py --suite evaluate
```

`run_evaluate_fortran.py` works by importing `evaluate.py`, replacing its
`bidiag_svd` symbol with a thin shim that serializes (d, e) to a binary file,
invokes `mr3gk_fortran/mr3gk_run` via subprocess, and reads the output back —
so the printed PASS/FAIL output and thresholds (res ≤ 7 nε, ortU/V ≤ 5 nε)
are bit-for-bit the same as `python3 evaluate.py` would print, but every
SVD goes through the pure-Fortran path.

## Test thresholds (from `evaluate.py`)

A test passes iff all three of these hold:
- residual `‖B − UΣVᵀ‖_max / (‖B‖·n·ε) ≤ 7.0`
- orthogonality `‖UᵀU − I‖_max / (n·ε) ≤ 5.0`
- orthogonality `‖VᵀV − I‖_max / (n·ε) ≤ 5.0`

## What's included

- `python_fortran/mr3gk_fortran/*.f90` — pure-Fortran orchestration layer
  (~1,400 lines: T_GK construction, splitting, sign matrices, zero-shift QR
  deflation, Bv recovery, Gram-Schmidt completion, σ rescaling).
- `python_fortran/mr3gk_fortran/build.sh` — compiles the orchestration
  layer and links it against `libxmr.so`.
- `python_fortran/dlaxre_gk.f` — patched copy of upstream `dlaxre.f`
  (Bug #1 fix: activate the GK-detection branch).
- `python_fortran/dlaxrb_clssfy_fix.f` — patched copy of upstream
  `dlaxrb_clssfy.f` (Bug #2 fix: AVGTHRESH=0 guard at deep tree nodes).
- `python_fortran/xmr_src/` — Paul Willems' XMR Fortran source, **untouched**.
- `python_fortran/run_evaluate_fortran.py` — driver that routes
  `evaluate.py` (and the other standard test harnesses) through the
  pure-Fortran executable.

The pure-Fortran driver shares the same `libxmr.so` kernel as the Python+Fortran
hybrid; the only difference is whether the bidiagonal-SVD orchestration layer
runs in Python (~640 lines of `mr3_gk.py`) or in Fortran
(~1,400 lines of `mr3gk_fortran/`). Both paths produce **bit-identical** σ
and U/V agreeing to ≤ 1 ULP across 21,375 specs (`docs/01_fortran_test_report.md`).

```bash

# Run the evaluation
python3 run_evaluate_fortran.py --suite evaluate

# Expected output: TOTAL: 379/379 passed
```
